# Request Intent Router

## 검색 키워드

request intent, prototype planning mode, design review mode, implementation mode, code review mode, planning no code, answer mode

## 목적

답변 전에 사용자 요청 의도를 분류하고, 맞는 답변 모드를 선택한다. 계획 요청에 전체 구현 코드가 나오거나, 구현 요청에 설계 원칙만 나오는 문제를 막는다.

## Prototype Planning Mode

다음 요청은 계획 모드다.

- "~ 같은 게임 만들고 싶은데 계획 세워줘"
- "프로토타입 계획 짜줘"
- "어떤 순서로 만들면 돼?"
- "2주 안에 검증할 수 있게 짜줘"
- "핵심 루프부터 잡아줘"

계획 모드에서는 전체 구현 코드를 작성하지 않는다.

출력해야 할 것:

1. 레퍼런스 해석
2. 핵심 재미 / 핵심 가설
3. 핵심 루프
4. 최소 기능 범위
5. Phase 별 일정
6. 책임 분리 초안
7. 상태 / 프로세스 SSOT 초안
8. 위험 요소
9. 이번 프로토타입에서 제외할 것
10. 첫 구현 단위

## Design Review Mode

다음 요청은 설계 검토 모드다.

- "이 설계 어때?"
- "구조 검토해줘"
- "책임 분리 봐줘"
- "문제점 찾아줘"

이 모드에서는 구현 코드를 작성하지 않는다. 책임, SSOT, API, 이벤트 위치, 위험 요소를 검토한다.

## Implementation Mode

다음처럼 사용자가 명시해야만 구현 코드를 작성한다.

- "코드 작성해줘"
- ".h/.cpp로 구현해줘"
- "컴파일 가능한 코드로 만들어줘"
- "Unreal C++ 전체 코드 줘"

## Code Review Mode

사용자가 코드를 붙여넣고 리뷰를 요청하면 코드 리뷰 모드다.

출력해야 할 것:

1. 컴파일 오류 가능성
2. API 사용 오류
3. 책임 분리 문제
4. 런타임 위험
5. 최소 수정안

## 핵심 규칙

계획 요청과 설계 검토 요청에서는 전체 구현 코드, 긴 `.h/.cpp`, compile-ready 코드 블록을 작성하지 않는다.
