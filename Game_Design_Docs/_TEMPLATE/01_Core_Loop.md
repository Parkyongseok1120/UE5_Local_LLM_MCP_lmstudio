# Core Loop

## Core Hypothesis

이 프로토타입이 가장 먼저 검증해야 하는 재미를 한 문장으로 적는다.

## 30-Second Loop

Observe -> Decide -> Act -> React -> Recover -> Repeat

## Must Have

- 핵심 입력 1~3개.
- 즉시 보이는 반응.
- 성공/실패 조건.
- 반복 가능한 테스트 씬.

## Should Have

- 핵심 감각을 강화하는 보조 피드백.
- 간단한 난이도 변화.

## Later

- 성장, 저장, 상점, 장비, 대형 맵, 멀티플레이, 복잡한 UI.

## First Implementation Slice

첫 구현 단위는 테스트 맵, 플레이어 1명, 대상 1~2개, 피드백 1개로 제한한다.
